分类型机器学习算法

分类问题是机器学习中最基础的任务类型之一。给定一组特征（自变量），模型需要预测样本属于哪个类别（应变量）。在量化交易中，分类算法可用于预测股票涨跌（涨=1/跌=0）、信用风险评估（违约=1/不违约=0）、 cancer诊断（恶性/良性）等场景。

σ 逻辑回归 (Logistic Regression)

逻辑回归虽然名字中有"回归"，但实际上是一种二分类算法。它通过 Sigmoid 函数将线性回归的输出映射到 (0, 1) 区间，表示样本属于正类的概率。

z = w₁x₁ + w₂x₂ + ... + wₙxₙ + b
P(y=1|x) = 1 / (1 + e⁻ᶻ)
若 P > 阈值(默认0.5)，预测为正类

核心思想：找到一组权重 w 和偏置 b，使得模型对训练数据的预测概率尽可能接近真实标签。通过梯度下降法最小化对数损失（Log-Loss）。

优点：

  • 优输出有概率意义，易于解释
  • 优训练速度快，适合大规模数据
  • 优不容易过拟合（正则化）

缺点：

  • 缺只能处理线性可分问题
  • 缺对特征相关性敏感

🌳 决策树 (Decision Tree)

决策树通过一系列if-else规则对数据进行递归划分，形成树状结构。每个内部节点是一个特征判断，叶子节点是类别标签。

基尼不纯度: Gini = 1 - Σ(pᵢ)²
信息增益: Gain = H(父) - Σ(|Dᵢ|/|D|) × H(子ᵢ)
选择使不纯度下降最多的特征分裂

分裂过程：从根节点开始，选择一个特征和阈值将数据分成两部分，使得分裂后子节点的纯度最高（Gini最小）。递归直到满足停止条件（最大深度、最小样本数等）。

优点：

  • 优直观易懂，可解释性强
  • 优可处理分类和数值特征
  • 优不需要特征标准化

缺点：

  • 缺容易过拟合
  • 缺对数据微小变化敏感

🌲 随机森林 (Random Forest)

随机森林是集成学习的代表算法，通过构建多棵决策树并投票来做出最终预测。"随机"体现在两方面：数据随机采样（Bagging）和特征随机选择。

1. 从训练集有放回采样 n 个子集
2. 每棵树随机选择 √m 个特征分裂
3. 每棵树独立生长，不剪枝
4. 预测时多数投票（分类）/平均（回归）

核心优势：通过集成多棵"弱学习器"（决策树），降低单棵树的方差，提高泛化能力。同时能输出特征重要性排序。

优点：

  • 优准确率高，抗过拟合
  • 优能处理高维数据
  • 优提供特征重要性评估

缺点：

  • 缺模型体积大，预测速度较慢
  • 缺可解释性不如单棵决策树

机器学习模型评价指标

训练好模型后，需要用合适的指标评估其性能。不同指标从不同角度衡量模型质量，选择正确的指标对于量化交易策略的评估至关重要。

📊 混淆矩阵 (Confusion Matrix)

混淆矩阵是一个 2×2 表格，展示模型预测与真实标签的四种组合，是所有分类指标的基础。

  • TN (真阴性): 实际恶性，预测恶性 ✓
  • TP (真阳性): 实际良性，预测良性 ✓
  • FP (假阳性): 实际恶性，预测良性 ✗
  • FN (假阴性): 实际良性，预测恶性 ✗
准确率 = (TP+TN) / (TP+TN+FP+FN)
精确率 = TP / (TP+FP)
召回率 = TP / (TP+FN)
F1 = 2 × (精确率×召回率) / (精确率+召回率)

📈 ROC 曲线 (Receiver Operating Characteristic)

ROC曲线以假阳性率(FPR)为横轴、真阳性率(TPR)为纵轴，展示模型在不同分类阈值下的性能表现。

TPR (真阳性率) = TP / (TP + FN)
FPR (假阳性率) = FP / (FP + TN)
阈值从 1 → 0 移动，计算每个阈值下的(TPR, FPR)

如何解读ROC曲线：

  • 曲线越靠近左上角，模型性能越好
  • 对角线(45°)代表随机猜测，AUC=0.5
  • 完美模型: AUC = 1.0 (曲线贴左上角)
  • 通常 AUC > 0.7 可用，> 0.8 良好，> 0.9 优秀

量化交易中的应用：ROC曲线可帮助找到最佳交易信号阈值——在最大化正确信号(TPR)的同时控制误信号(FPR)。

🎯 AUC (Area Under Curve)

AUC是ROC曲线下的面积，取值范围 [0, 1]，是一个阈值无关的综合性指标。

AUC = ∫₀¹ TPR(FPR) d(FPR)
≈ 梯形法数值积分

AUC的物理含义：

  • AUC = 随机取一个正样本和一个负样本，模型给正样本打分高于负样本的概率
  • AUC = 0.5: 模型无判别能力（等同于抛硬币）
  • AUC = 1.0: 完美分类（所有正样本得分 > 所有负样本得分）
  • AUC < 0.5: 模型比随机还差（可反转预测）

为什么用AUC？AUC不依赖于具体阈值，能公平比较不同模型在整体概率排序上的能力，是金融风控和量化策略评估的首选指标。

指标之间的关系与权衡

在实际应用中，不同指标之间往往存在权衡关系。例如在量化交易中：

  • 精确率 vs 召回率：预测股票上涨时，高精确率=少发错信号，高召回率=不漏真信号。需要根据交易成本权衡。
  • F1分数：精确率和召回率的调和平均，适合类别不平衡场景。
  • AUC的优势：不受阈值和类别不平衡影响，是最稳健的模型对比指标。
  • 在癌症诊断中：召回率(不漏诊)通常比精确率更重要；在交易中，精确率(不亏钱)通常更重要。

结论与分析

  • 逻辑回归表现最优：AUC高达0.9981，说明乳腺癌数据集的特征与目标之间接近线性可分，标准化后的逻辑回归能够完美拟合。
  • 随机森林次之：AUC=0.9893，召回率最高(0.9626)，说明它在识别良性肿瘤方面表现突出，且提供了有价值的特征重要性排序。
  • 决策树相对较弱：AUC=0.9441，单棵决策树容易过拟合，泛化能力不如集成方法和正则化的逻辑回归。
  • 最重要特征：worst concave points（最差凹点）和 worst area（最差面积）是区分良恶性最关键的指标。
  • 量化交易启示：在特征与目标关系接近线性时，简单模型(逻辑回归)可能优于复杂模型；集成方法在特征交互复杂时更有优势。

Python 实现代码

以下代码使用 scikit-learn 库实现完整的机器学习分类流程：数据加载 → 数据划分 → 模型训练 → 模型评估 → ROC曲线绘制。

import numpy as np
import matplotlib.pyplot as plt
from sklearn.datasets import load_breast_cancer
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression
from sklearn.tree import DecisionTreeClassifier
from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import (
    roc_curve, auc, confusion_matrix,
    accuracy_score, precision_score, recall_score, f1_score
)

# 1. 加载乳腺癌数据集
data = load_breast_cancer()
X, y = data.data, data.target
print(f"样本数: {X.shape[0]}, 特征数: {X.shape[1]}")
print(f"类别分布: 良性={sum(y==1)}, 恶性={sum(y==0)}")

# 2. 数据划分 (训练集70%, 测试集30%)
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.3, random_state=42, stratify=y
)

# 标准化 (逻辑回归需要)
scaler = StandardScaler()
X_train_s = scaler.fit_transform(X_train)
X_test_s = scaler.transform(X_test)

# 3. 构建并训练分类模型
models = {
    '逻辑回归': LogisticRegression(max_iter=1000, random_state=42),
    '决策树': DecisionTreeClassifier(max_depth=5, random_state=42),
    '随机森林': RandomForestClassifier(
        n_estimators=100, max_depth=5, random_state=42),
}

results = {}
for name, model in models.items():
    # 逻辑回归用标准化数据，树模型用原始数据
    if name == '逻辑回归':
        model.fit(X_train_s, y_train)
        y_prob = model.predict_proba(X_test_s)[:, 1]
    else:
        model.fit(X_train, y_train)
        y_prob = model.predict_proba(X_test)[:, 1]

    y_pred = (y_prob >= 0.5).astype(int)

    # 4. 评估模型
    fpr, tpr, _ = roc_curve(y_test, y_prob)
    auc_val = auc(fpr, tpr)
    cm = confusion_matrix(y_test, y_pred)

    print(f"\n{name}:")
    print(f"  准确率: {accuracy_score(y_test, y_pred):.4f}")
    print(f"  AUC:    {auc_val:.4f}")
    print(f"  混淆矩阵:\n  {cm}")

    results[name] = {'fpr': fpr, 'tpr': tpr, 'auc': auc_val}

# 5. 绘制ROC曲线
plt.figure(figsize=(8, 6))
colors = ['#e74c3c', '#2ecc71', '#3498db']
for (name, res), color in zip(results.items(), colors):
    plt.plot(res['fpr'], res['tpr'],
             color=color, linewidth=2.5,
             label=f'{name} (AUC={res['auc'']:.4f})')

plt.plot([0, 1], [0, 1], 'k--', label='随机分类器')
plt.xlabel('假阳性率 (FPR)')
plt.ylabel('真阳性率 (TPR)')
plt.title('ROC 曲线对比')
plt.legend()
plt.grid(True, alpha=0.3)
plt.tight_layout()
plt.savefig('roc_curves.png', dpi=150)
plt.show()
